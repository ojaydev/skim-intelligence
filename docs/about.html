Skim Intelligence is a five-agent system powered by Claude Opus 4.7 that autonomously scans live prediction market orderbooks, reasons about structural yield opportunities, paper-trades approved signals, and attributes performance at every epoch close.

It makes no directional bets. Every edge it extracts — maker rebates, liquidity rewards, mint/burn arbitrage — is available regardless of whether markets resolve YES or NO. The intelligence layer of Skim, a liquidity engine for African prediction markets.

Core Insight
Prediction markets pay bots to exist. Maker rebates, liquidity rewards, and mint/burn arbitrage are structural edges baked into platform design — available regardless of whether markets resolve YES or NO. Most operators access these edges through hardcoded rules. Skim uses Opus 4.7 to reason about each market situation dynamically, the way a sophisticated quantitative analyst would.
Three Strategy Layers
01
Mint / Burn Arbitrage
A YES share + NO share = $1.00 by protocol invariant. When the complement sum drifts above or below $1.00 by more than the taker fee, a risk-free arb exists. The Alpha Agent checks both legs simultaneously and hard-rejects if the snapshot is stale (>60s old).
net margin > 3.2% @ P>0.70
02
CLOB Market Making
Post two-sided limit orders at ±3¢ from mid, capturing the bid-ask spread on fills. Zero maker fees on Bayse. Inventory neutrality enforced — YES and NO positions must stay within ±15% of each other by notional to prevent directional drift.
spread > 3% · depth > $200
03
Liquidity Reward Farming
Bayse pays from a fixed pool per market for resting two-sided orders. Market-making orders automatically qualify when two_sided_eligible = true. Incremental yield stacked on top of spread capture — no additional capital deployed.
pool > $50 · est. +1.2% yield
Taker fee schedule drives arb thresholds: P = 0.30–0.50 → 5–7% fee (need 6%+ gross); P = 0.50–0.70 → 3–5% fee (sweet spot for MM); P > 0.70 → 3% floor (cheapest arb zone). All three layers are evaluated simultaneously on every market snapshot.

Scanner
Deterministic
Trigger: WebSocket book delta (Bayse + Polymarket)
Holds persistent outgoing WS connections to relay.bayse.markets and ws-subscriptions-clob.polymarket.com. Runs inside a Cloudflare Durable Object so connections survive across requests. On each book delta event, computes derived metrics: mid price, spread %, bid/ask depth, complement sum, reward eligibility, data freshness. Writes MarketSnapshot to KV at key market:{id}:snapshot.
Alpha
claude-opus-4-7
Trigger: Per market snapshot (parallel, max 10 concurrent)
The core intelligence and the demo centrepiece. Receives the full market snapshot plus current portfolio state and reasons through all three strategy layers simultaneously. Outputs a structured JSON signal with opportunity_score (0–1), per-strategy action plans, risk_flags, and the full thinking text — streamed live to the dashboard as it generates. This raw reasoning is intentionally exposed: legible, auditable intelligence.
Risk
claude-opus-4-7
Trigger: Per Alpha signal where recommendation = "enter"
The circuit breaker. Enforces hard limits — daily loss cap, per-market exposure ceiling, total portfolio exposure, data freshness (hard-rejects if snapshot age > 60s), inventory imbalance above 20%, execution mode guard. Applies soft modifications — halves notional on low-confidence signals, downgrades near-resolution markets. Returns approved / modified / rejected with full reasoning.
Execution
Paper Engine
Trigger: Per approved or modified Risk decision
Pure state machine — no LLM. Simulates order placement using a realistic fill model: maker orders fill probabilistically based on volume/depth ratio (max 85%); taker arb orders fill at 95% within top-of-book, 60% at second level, with ±0.2% slippage. Writes orders, fills, and positions to Cloudflare D1. Runs inventory rebalance checks every 60s. Closes epochs every 5 minutes.
Reporter
claude-opus-4-7
Trigger: Every 5-minute epoch close — receives full P&L breakdown from Execution
Generates a plain-English attribution summary bucketed into spread capture / liquidity rewards / arb profits / fees paid, then produces the shareable epoch card data. Hard rules: always label results as "paper trading"; use "estimated" for unconfirmed reward income; never hide negative epochs. Output feeds the Share Card component and the epoch history in D1.

Orchestrator

Cloudflare Durable Object. Coordinates the full pipeline, holds WebSocket connections to the frontend, maintains cycle state. Survives across requests — state persists in DO Storage.

Execution Mode

Toggled via POST /api/admin/mode. Options: observe (no fills) · paper (simulated) · live_limited. Always default to paper in development.

WebSocket Data Layer
Polymarket
wss://ws-subscriptions-clob.polymarket.com/ws/market
Public · no auth required · events: book, price_change, best_bid_ask, market_resolved · PING every 10s required.
Bayse
wss://relay.bayse.markets/ws/market-data
HMAC-SHA256 auth with BAYSE_API_KEY / BAYSE_API_SECRET · orderbook snapshots, activity feed, price updates. Docs at docs.bayse.markets.
Dashboard
wss://skim.workers.dev/api/ws
Cloudflare Durable Object hibernation API · broadcasts signals, fills, risk decisions, epoch closes. Requires binaryType = "arraybuffer" post-March 2026.
Bootstrap pattern: On connect, fetch the initial orderbook via REST (GET /book?token_id=<id>), then subscribe and apply incoming book delta messages. Re-fetch on reconnect — Polymarket has no sequence numbers so deltas cannot be replayed. Bayse similarly requires a snapshot on reconnect.

Cloudflare limits: Outgoing WS connections (Scanner → Bayse/Polymarket) do not support hibernation — the Scanner DO must stay resident. Incoming dashboard connections use ctx.acceptWebSocket() for hibernation. Max message size 32 MiB. Idle timeout 100s on Free/Pro — keepalive pings required.
Data Model (D1 / KV)
-- Core tables in Cloudflare D1 (SQLite) CREATE TABLE signals ( id TEXT PRIMARY KEY, market_id TEXT NOT NULL, opportunity_score REAL, recommendation TEXT, -- enter | observe | skip thinking TEXT, -- full Alpha reasoning reasoning_summary TEXT, strategies_json JSON, created_at TEXT ); CREATE TABLE risk_decisions ( id TEXT PRIMARY KEY, signal_id TEXT REFERENCES signals(id), decision TEXT, -- approved | modified | rejected hard_limit_triggered TEXT, modifications_json JSON ); CREATE TABLE paper_fills ( id TEXT PRIMARY KEY, order_id TEXT REFERENCES paper_orders(id), fill_price REAL, fill_notional_usd REAL, fee_usd REAL, slippage_usd REAL, filled_at TEXT ); CREATE TABLE epoch_reports ( id TEXT PRIMARY KEY, headline TEXT, attribution_json JSON, -- spread/rewards/arb/fees narrative TEXT, share_card_json JSON, net_pnl_usd REAL );
KV keys: market:{id}:snapshot (latest orderbook state) · orchestrator:state · portfolio:state · rate:alpha:{market_id} (1 Alpha call per market per 2 min).

Hackathon Context

Built in 5 days for the Anthropic Opus 4.7 Hackathon. Submission deadline: April 26, 2026, 8 PM EST. Theme: "Build From What You Know." Skim Intelligence is the intelligence layer of Skim — a Bayse-native liquidity engine for African prediction markets currently in development.

The demo centres on the Alpha Agent's live reasoning stream. Opus 4.7 analyses each market out loud — assessing spread quality, arb viability, inventory balance, and reward yield in real time. That transparency is the product differentiator: not a black box, but legible, auditable intelligence that a user can inspect and trust.

The five-day build sequence: Day 1 — live market data via WS; Day 2 — Alpha Agent reasoning in UI; Day 3 — paper trading end-to-end; Day 4 — dashboard polish + share card; Day 5 — submission. Licensed MIT. All agent prompts, orchestration logic, and paper trading engine published open source at submission.

Post-Hackathon Roadmap

This codebase is not a throwaway. The paper trading engine becomes the PAPER mode from Skim PRD v2.1 — the shadow-mode test harness for Milestone A. Each component maps directly to a production use case.

Alpha Agent
→
Core reasoning engine for real execution decisions on live Bayse positions. Prompt and strategy config become user-tunable parameters.
Risk Agent
→
Production circuit breaker with real position limits, user-specific daily loss caps, and regulatory compliance rules for Nigerian market operations.
Execution Agent
→
Replace paper order simulation with live Bayse API calls. Paper mode becomes the shadow-mode test harness — runs alongside live execution for comparison.
Reporter Agent
→
User-facing P&L statements and daily share cards for Skim account holders. Attribution methodology documented publicly as the trust surface.
Orchestrator
→
Always-on Durable Object with Cron Triggers for guaranteed cycle execution. Add user session management and per-user strategy isolation.
D1 Schema
→
Extended with users, deposits, ledger, withdrawals, and Paystack integration for NGN/USD flows. Open source repo becomes the public trust surface.
